AXES FAQ

What’s new?

What can I find through the AXES system? AXES allows you to directly search in audiovisual material with text and visual queries (“similarity search”). It is a search beyond metadata search.

What can be found?

The system can find people categories (e.g. a woman) or individuals (Angela Merkel); location categories (countryside) or specific places (Eifeltower); object categories (car) or specific objects (Mercedes logo or Italian flag); and event categories (mountain climbing).

What are the key features?

  • The archive can be searched independent from available metadata: AXES is very useful for material which is poorly annotated, because it creates annotations on its own (ASR, Names, Persons, Places, Objects)
  • AXES combines textual and visual search
  • AXES enables visual search (including similarity search) - whereas most digitised collections so far only let us search in texts.
  • The visual search is language independent (“query by example”)
  • AXES enables a keyframe based search. Results are connected to relevant images or sequences (no need of fast forwarding any more).
  • AXES let us search for events
  • AXES automatically suggests related content

What are pretrained objects?

Popular terms like house, cat or car were already “trained” into the system. Searches for these terms are much faster than “unknown” queries.

Does the system learn?

New terms are learned on-the-fly. Once computed, they're stored in the cache, so next time they need not be recomputed again. So, the same search will be much faster. (see also: Pretrained Objects)

What’s not in there?

Some features were not in the specific focus of the AXES project. These are:

  • multi keyword searches
  • language independency in text searches
